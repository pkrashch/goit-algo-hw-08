# ДЗ №8: Мінімальна Вартість З'єднання Кабелів (Алгоритм Купи)

Цей проєкт розв'язує класичну задачу мінімізації загальних витрат при послідовному з'єднанні мережевих кабелів різної довжини.

## Задача

Дано набір кабелів із різними довжинами. Об'єднувати їх потрібно по два за раз. Витрати на об'єднання дорівнюють сумі довжин цих двох кабелів. Мета — знайти порядок об'єднання, який мінімізує загальну суму всіх витрат.

## Алгоритм Рішення: Міні-Купа (Min-Heap)

Для досягнення мінімальних загальних витрат на кожному кроці необхідно об'єднувати **два найменші доступні кабелі**. Це гарантує, що найчастіше використовуватимуться короткі кабелі, а найдовші кабелі будуть додані лише один раз на пізніх етапах.

Цей підхід ідеально реалізується за допомогою **Мінімальної Купи (Min-Heap)**, яка виконує операції вилучення мінімального елемента ($O(\log N)$) максимально швидко.

### Покроковий процес:

1.  **Ініціалізація:** Усі початкові довжини кабелів додаються до мінімальної купи.
2.  **Об'єднання:** Поки в купі є більше одного кабелю:
    * Витягуються два найменші елементи ($C_1$, $C_2$).
    * Вартість $C_1 + C_2$ додається до загальних витрат.
    * Новий кабель довжиною $C_1 + C_2$ повертається назад у купу.
3.  **Результат:** Фінальна накопичена сума є мінімально можливою.

## Аналіз Складності

* **Структура даних:** Міні-Купа (`heapq` у Python).
* **Часова складність:** $\mathbf{O(N \log N)}$, де $N$ — початкова кількість кабелів.
    * $N-1$ операцій об'єднання. Кожна операція включає дві дії $O(\log N)$ (витягування та вставка).

## Приклад Тестування

**Вхід:** `[4, 3, 2, 6]`
**Процес:**
1. (2 + 3) = 5. Витрати: 5. Heap: [4, 5, 6]
2. (4 + 5) = 9. Витрати: 9. Heap: [6, 9]
3. (6 + 9) = 15. Витрати: 15. Heap: [15]

**Мінімальні Загальні Витрати:** $5 + 9 + 15 = \mathbf{29}$
